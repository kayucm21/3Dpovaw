## 🛠️ Быстрая установка панели

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/kayucm21/3Dpovaw/main/install.sh)
```

### Выбор способа доступа к панели

Во время установки будет вопрос:

```text
1) Через Nginx на порту 8080
2) Напрямую на порту 3000
3) Через Nginx с доменом + HTTPS
```

Можно выбрать вручную в установщике, либо заранее указать выбором командой:

```bash
ACCESS_MODE=1 bash <(curl -fsSL https://raw.githubusercontent.com/kayucm21/3Dpovaw/main/install.sh)
ACCESS_MODE=2 bash <(curl -fsSL https://raw.githubusercontent.com/kayucm21/3Dpovaw/main/install.sh)
ACCESS_MODE=3 bash <(curl -fsSL https://raw.githubusercontent.com/kayucm21/3Dpovaw/main/install.sh)
```

## 📋 Требования к серверу

- Ubuntu 22.04 / 24.04 или Debian 11 / 12
- Поддомен с A-записью на IP сервера (например `naive.yourdomain.com`)
- Открытые порты: 22, 80, 443, 3000
- Минимум 1 GB RAM (для сборки Caddy — временно нужно 512 MB)

## 🔄 Процесс установки протокола (автоматически)

1. Выбор протокола: `Naive` или `VLESS`
2. Обновление системы и зависимостей
3. Включение BBR (алгоритм TCP от Google)
4. Настройка фаервола UFW
5. Для Naive: установка Go и сборка Caddy с `forwardproxy`
6. Сохранение домена, порта и пользователя в панели
7. Генерация готовой ссылки подключения

## 🎯 Быстрый старт после установки

1. **Войдите в панель** по адресу `http://YOUR_SERVER_IP:3000`
   - Логин по умолчанию: `admin` / `admin`
   - **Сразу смените пароль!**

2. **Настройте 2FA** (рекомендуется):
   - Перейдите в раздел "Безопасность"
   - Нажмите "Включить 2FA"
   - Отсканируйте QR-код через Google Authenticator или Authy
   - Введите 6-значный код для подтверждения

3. **Установите протокол**:
   - Перейдите в раздел "Установка"
   - Выберите NaiveProxy или VLESS
   - Введите домен и email
   - Нажмите "Начать установку"

4. **Добавьте пользователей**:
   - Перейдите в раздел "Пользователи"
   - Нажмите "Добавить"
   - Введите имя пользователя и пароль
   - Скопируйте ссылку для подключения

5. **Настройте мониторинг**:
   - Перейдите в раздел "Ресурсы"
   - Наблюдайте за CPU, RAM, диском и сетью в реальном времени

6. **Создайте бэкап**:
   - Перейдите в раздел "Настройки"
   - Нажмите "Создать бэкап"
   - Бэкап автоматически сохранится в `backups/`

## 🔧 Управление через API

Панель предоставляет REST API для автоматизации:

### Мониторинг
```bash
GET /api/resources          # CPU, RAM, диск, сеть в реальном времени
GET /api/connections        # История подключений
GET /api/analytics/top-users # Топ пользователей
```

### Безопасность
```bash
GET /api/security/twofa         # Статус 2FA
POST /api/security/twofa/setup  # Инициализация 2FA
POST /api/security/twofa/verify # Подтверждение 2FA
```

### Бэкапы
```bash
POST /api/backup   # Создать бэкап
POST /api/restore  # Восстановить из бэкапа
```

### Пользователи
```bash
GET /api/users     # Список пользователей
POST /api/users    # Добавить пользователя
DELETE /api/users  # Удалить пользователя
```

### Устройства
```bash
GET /api/devices       # Список устройств
POST /api/devices/block    # Заблокировать устройство
POST /api/devices/unblock  # Разблокировать устройство
```

### Зависимости Node.js
```json
{
  "express": "^4.18.2",
  "express-session": "^1.17.3",

  "cors": "^2.8.5",

  "qrcode": "^1.5.3"
```

### Файлы данных
- `panel/data/config.json` - основная конфигурация
- `panel/data/users.json` - пользователи панели
- `panel/data/devices.json` - устройства и блокировки
- `panel/data/connections-log.json` - история подключений
- `panel/data/twofa.json` - настройки 2FA
- `panel/backups/` - резервные копии

### Port & Access
- Порт по умолчанию: `3000`
- Доступ через: `http://YOUR_IP:3000`
- HTTPS: через Nginx с доменом
